1. Jumeau Numérique (Oscilloscope Virtuel)

Objectif : Utiliser le composant matériel Timer (TIM3) pour générer un signal carré à rapport cyclique variable (PWM) afin de faire varier la luminosité de la LED sur PA5 sans bloquer le processeur.


Rapport Cyclique : 50% | CCRx : 500 / 1000

LED Dimmable

Oscilloscope (CH1)

Code C : Modification dynamique de la PWM

Contrairement aux délais logiciels (HAL_Delay), la PWM est générée par le matériel. Le CPU est libre de faire autre chose.

// 1. Démarrage du signal PWM sur le Timer 3, Canal 2 (Lié à PA5)
HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_2);

// Boucle infinie - main.c
while (1) 
{ 
    // Exemple : Augmenter progressivement la luminosité (Fading)
    for(uint16_t duty = 0; duty <= 1000; duty += 10) 
    {
        // __HAL_TIM_SET_COMPARE modifie le registre CCR (Capture/Compare Register)
        // C'est ce registre qui dicte la largeur de l'impulsion (Duty Cycle)
        __HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_2, duty);
        
        HAL_Delay(10); // Vitesse de la transition visuelle
    }
}

2. Post-Lab : Ingénierie des Timers

Qu'est-ce que la PWM (Modulation de Largeur d'Impulsion) et quel est l'intérêt de la composante matérielle Timer par rapport à une boucle logicielle (HAL_Delay) ?

La PWM est une technique consistant à créer un signal carré dont on modifie le temps à l'état HAUT (Ton) par rapport à la période totale (T). Ce ratio est appelé Rapport Cyclique (Duty Cycle). En jouant sur ce ratio à haute fréquence, on fait varier la valeur moyenne de la tension (ex: 50% = 1.65V moyen au lieu de 3.3V).

L'intérêt du Timer : Si on créait un signal PWM avec des HAL_Delay() et des HAL_GPIO_WritePin(), le microcontrôleur serait bloqué à 100% dans ces boucles d'attente (Bit-banging). En utilisant le périphérique matériel Timer, le processeur configure simplement les registres (Fréquence, Duty cycle) puis passe à autre chose. Le composant Timer génère ensuite le signal carré de manière totalement autonome en arrière-plan, libérant 100% du CPU pour l'ADC ou l'I2C.

Le STM32C0 tourne à une fréquence système de 48 MHz. Comment calculer le "Prescaler (PSC)" et "l'Auto-Reload Register (ARR)" pour obtenir un signal PWM avec une fréquence de 1 kHz (1000 Hz) et une résolution de 1000 pas ?

La formule fondamentale de la fréquence d'un Timer STM32 est :

F_PWM = F_CPU / ((PSC + 1) × (ARR + 1))

Étape 1 : Choix de la résolution (ARR). On veut pouvoir régler la luminosité de 0 à 1000 (100.0%). On fixe donc ARR = 999 (car le compteur compte à partir de 0, donc 999+1 = 1000).

Étape 2 : Calcul du diviseur (Prescaler). On réarrange la formule :
(PSC + 1) = F_CPU / (F_PWM × (ARR + 1))
(PSC + 1) = 48 000 000 / (1000 × 1000)
(PSC + 1) = 48
Donc, PSC = 47.

Conclusion pour CubeMX : On entre 47 dans le champ Prescaler et 999 dans le champ Counter Period (ARR).

À quoi sert la macro __HAL_TIM_SET_COMPARE() ?

Le registre ARR définit la Période Totale (la fréquence de notre signal). Il ne change jamais en cours de route.
Le registre CCR (Capture/Compare Register), lui, définit à quel moment le signal doit redescendre à ZÉRO (le rapport cyclique).

La macro __HAL_TIM_SET_COMPARE() sert à écrire directement une nouvelle valeur dans le registre CCR à la volée. Si ARR vaut 1000, et qu'on utilise __HAL_TIM_SET_COMPARE(&htim, channel, 250), le signal restera à l'état HAUT pendant 25% du temps (250/1000), ce qui allumera la LED à 25% de sa puissance maximale.
